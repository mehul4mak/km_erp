How to read your documents

Those reference numbers aren't random — they're the map.

Every order, job, receipt and delivery gets a reference the system creates on its own. Once you can read one, you can follow any product from a customer order all the way to the finished box — and the dashboard is just a live tally of all of them.

Part 1 · The anatomy

Reading a reference number

Take a real one from your system — the job that builds the motor:

WH/ MO/ 00002
WH — the plant

Which warehouse it belongs to. You have one: your Delhi NCR unit.

MO — the type

What kind of document. MO = Manufacturing Order (a job on the floor).

00002 — the count

The running number. This is the 2nd manufacturing job ever created.

That's the whole trick. The system keeps a separate counter per document type and stamps the next number automatically — you never type it. Here's the full family you'll see:

Looks likeWhat it isBorn when…
S00001Sales order / quotationyou raise a quote for a customer
P00001Purchase order to a vendorcomponents need buying
WH/MO/00001Manufacturing Order — a production joban order needs something built
WH/IN/00001Goods Receipt (incoming) — the GRNa purchase order is confirmed
WH/OUT/00001Delivery (outgoing) to the customeran order is ready to ship
OP/00023Internal move — stock shifting inside the plantmaterial moves between locations

IN = inward, OUT = outward, MO = make, OP = internal operation. Read left to right: plant / type / number.

The one field that connects everything

Every document has an Origin — a plain-language "where did this come from." The motor job's origin is WH/MO/00001 (the mixer job that needs it). The receipt's origin is P00001. The delivery's origin is S00001. Follow the Origin trail and you can walk from the finished box all the way back to the customer's order — that's full traceability, built in.

Behind the scenes the system also groups the whole chain under one invisible thread (a "procurement group", here named after S00001) so it always knows these documents belong together — which is how the Order page can list a job's siblings.

Part 3 · The dashboard

The dashboard is a live tally of those documents

Nothing on the dashboard is entered by hand. Each tile is the system counting one type of document in real time — so it always shows what's actually going on right now:

Confirmed Order Value
◂ sums confirmed S##### orders

Total value of customer orders that are confirmed — money committed on the floor.

Quotes in Pipeline
◂ counts draft/sent S##### quotes

Quotations not yet confirmed — your sales pipeline.

Jobs on the Floor
◂ counts open WH/MO/##### jobs

Manufacturing orders in progress — what production is working on.

Purchases Pending
◂ counts unconfirmed P##### orders

RFQs waiting to be confirmed into purchase orders.

Items to Replenish
◂ counts components below their min

Parts at or under their reorder point — buy these before you run out.

Recent Orders / On the Floor
◂ latest S##### and WH/MO/#####

The two live lists — newest customer orders and current jobs, click through to the document.

How to use it

Read the dashboard top-left to bottom-right: is money coming in (order value, pipeline), is the floor busy (jobs), are we buying what we need (purchases, replenish). When a number looks wrong, click into that list — every tile drills straight down to the documents behind it.
