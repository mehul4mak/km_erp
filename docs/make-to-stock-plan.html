Item 7 · make only the shortfall

Netting demand against stock — the logic, before we build it.

Today a confirmed sale always creates a manufacturing job, even if the mixers are already on the shelf. This is the plan to fix that: reserve from stock first, and make or buy only what's genuinely short — the classic ERP idea of net requirements.

The one rule

Net requirements

Every decision below comes from a single formula, applied at each level of the BOM:

Net = Ordered quantity  −  Free in stock  −  Already on order
    If Net ≤ 0 → reserve from stock, make nothing
    If Net > 0 → make / buy the Net only

Read it in one line

Ship from stock where you can; drop to a job only for the shortfall; that job pulls components the same way — reserve if in store, otherwise a sub-job (semi-finished) or a purchase (bought-in). The dotted arrow is the recursion: a sub-assembly shortfall runs the same test on its parts.

Why it changes

Today vs proposed

✗ Today — make-to-order

Always manufactures

  • Every finished good carries the MTO ("replenish on order") route
  • MTO means "make the full ordered quantity" — it never looks at stock
  • So 10 ordered = an MO for 10, even if 10 are on the shelf
  • Great for the cascade demo; wrong for a real factory with stock
✓ Proposed — net requirements

Makes only the gap

  • Finished goods reserve from stock first
  • An MO is created only for the shortfall (Ordered − Free)
  • Enough stock → no job at all, ships straight from the store
  • Components net the same way — PO only for what's short

How we'd build it

The change, in five moves

Drop the MTO route from finished goods and semi-finished sub-assemblies.

Removes the "always make the full qty" behaviour. Keep the Manufacture and Buy routes.

Add a small planning addon (mfg_planning) that extends sale.order.action_confirm.

On confirm, for each manufacturable line: read free stock, reserve the available part, and create an MO for the shortfall only — then let it cascade.

Net the components too inside that MO.

The job reserves in-store components; raises a sub-MO for a short semi-finished, or a PO for a short bought-in part. This is Odoo's reservation already — we just stop MTO from over-ordering.

Show the split on the order.

Add a line to the order page: "Reserved from stock: 4 · To manufacture: 6" so the netting is visible, not hidden.

Re-verify the golden demo path.

Confirm the flagship order still cascades correctly when stock is empty — the behaviour must be identical to today at zero stock.

One judgement call

Two ways to handle the shortfall: (A) a custom confirm hook that makes the shortfall immediately (recommended — matches "manufacture to fulfil this order"), or (B) pure make-to-stock via min/max reorder rules (simpler, but the sale waits for the next stock batch and decouples order from job). I recommend A.

Risk to watch

Changing routes touches the cascade the whole demo rests on. The mitigation is test case 1: at zero stock the behaviour must be byte-for-byte what it is today. We build behind that guarantee, then layer the stock-netting on top.
